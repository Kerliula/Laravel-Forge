# Laravel Forge

Dockerized Laravel backend template with JWT authentication, MariaDB, Redis, and Mailpit. No frontend -- pure API only.

The goal of this template is to let you start a new project with a single command, requiring changes only to the
.env.example file.

## Stack

Before running `make init`, check the links below for the latest versions and update `.env.example` as needed.

| Service | Default Version | Latest releases                                                           |
|---------|-----------------|---------------------------------------------------------------------------|
| PHP-FPM | 8.5 (Alpine)    | https://www.php.net/releases/                                             |
| Nginx   | 1.30 (stable)   | https://nginx.org/en/download.html                                        |
| MariaDB | 12.3            | https://mariadb.com/docs/release-notes/community-server                   |
| Redis   | 8.6             | https://redis.io/docs/latest/operate/oss_and_stack/install/install-redis/ |
| Mailpit | v1.29           | https://github.com/axllent/mailpit/releases                               |

> **Note:** Nginx uses even minor numbers for stable releases (e.g. 1.30) and odd for mainline (e.g. 1.31). Prefer
> stable for templates.

## Prerequisites

- Docker & Docker Compose
- Make
- Git

## Usage

### Option A — degit (recommended)

[degit](https://github.com/Rich-Harris/degit) copies the repository files without the `.git` history, so your new
project starts clean.

```bash
npx degit Kerliula/laravel-forge my-project
cd my-project
make init
```

### Option B — clone and detach

```bash
git clone https://github.com/Kerliula/laravel-forge.git my-project
cd my-project
rm -rf .git
git init
make init
```

## Quick Start

```bash
make init
```

The API will be available at `http://localhost:8080`.
Mailpit UI at `http://localhost:8025`.

## Commands

### Lifecycle

| Command        | Description                           |
|----------------|---------------------------------------|
| `make init`    | Full setup from scratch (destructive) |
| `make up`      | Start containers                      |
| `make down`    | Stop containers                       |
| `make down-v`  | Stop containers and remove volumes    |
| `make build`   | Rebuild images                        |
| `make restart` | Restart containers                    |
| `make ps`      | List running containers               |
| `make logs`    | Tail container logs                   |

### Database

| Command              | Description                    |
|----------------------|--------------------------------|
| `make migrate`       | Run migrations                 |
| `make seed`          | Run seeders                    |
| `make fresh`         | Drop all tables and re-migrate |
| `make migrate-fresh` | Fresh migration with seeders   |
| `make tinker`        | Open Laravel Tinker REPL       |
| `make db`            | Open MariaDB shell             |

### Code Quality

| Command           | Description                 |
|-------------------|-----------------------------|
| `make lint`       | Check code style (dry run)  |
| `make fix`        | Auto-fix code style         |
| `make analyse`    | Run PHPStan static analysis |
| `make baseline`   | Generate PHPStan baseline   |
| `make ide-helper` | Generate IDE helper files   |

### Testing

| Command                        | Description                  |
|--------------------------------|------------------------------|
| `make test`                    | Run tests in parallel        |
| `make test-coverage`           | Run tests with HTML coverage |
| `make test-filter filter=Name` | Run filtered tests           |
| `make test-suite suite=Name`   | Run specific test suite      |

### Utilities

| Command                | Description              |
|------------------------|--------------------------|
| `make artisan cmd=...` | Run any Artisan command  |
| `make bash`            | Shell into app container |
| `make telescope-clear` | Clear Telescope entries  |
